# Submission assets and runbook

## Objective

Prepare the final demo and documentation package so the project is easy to judge and easy to replay.

## Background

A technically good project can still underperform if the setup and narrative are unclear. This issue ensures the output is submission-ready.

## Scope

- end-to-end runbook
- demo recording checklist
- short architecture summary
- tx proof collection
- screenshots for approved and blocked paths
- final README polish

## Dependencies

- all MVP implementation issues complete or intentionally deferred

## Acceptance criteria

- a teammate can replay the demo from the runbook
- demo recording can be completed without improvising technical steps
- README clearly explains the product and architecture
- approved payment and blocked payment evidence are both captured

## Out of scope

- investor deck
- long-form product memo
- post-hackathon roadmap deck
